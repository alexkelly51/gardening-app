# Gardening Tracker вЂ” Build Plan

## Overview

A two-view app for managing your garden:

1. **Plant Catalog** вЂ” browse, search and filter all your plants
2. **Weekly To-Do** вЂ” smart task list showing what needs doing this week, with completion tracking

## Data Model

All data lives in a Google Sheet with three tabs. You can edit plants and care tasks directly in the sheet at any time вЂ” the app reads live from it.

### Tab 1: Plants

| Column         | Notes                                            |
| -------------- | ------------------------------------------------ |
| id             | Unique identifier (e.g. 1, 2, 3)                 |
| name           | e.g. "Lavender"                                  |
| variety        | e.g. "Hidcote" (optional)                        |
| quantity       | How many you have                                |
| position       | Full Sun / Partial Shade / Full Shade            |
| date_purchased | DD/MM/YYYY (optional)                            |
| price          | Number (optional)                                |
| notes          | Free text (optional)                             |
| image_filename | e.g. lavender.jpg вЂ” served from backend /images/ |

### Tab 2: CareTasks

| Column         | Notes                                              |
| -------------- | -------------------------------------------------- |
| id             | Unique identifier                                  |
| plant_id       | Matches id in Plants tab                           |
| action         | Feed / Prune / Propagate / Other                   |
| detail         | e.g. "Use tomato feed at half strength" (optional) |
| month_start    | 1вЂ“12                                               |
| month_end      | 1вЂ“12                                               |
| frequency_days | Leave blank for one-off per season                 |

### Tab 3: TaskLog

| Column       | Notes                        |
| ------------ | ---------------------------- |
| id           | Auto-generated by app        |
| plant_id     |                              |
| care_task_id |                              |
| completed_at | ISO datetime, written by app |

> TaskLog is written by the app when you tick off a task. You don't need to edit it manually, but you can view or clear entries directly in the sheet.

## Scheduling Logic

A task is **due** when:

- Current month is within `month_start`вЂ“`month_end`
- AND either: never completed this season, OR `last_completed_at + frequency_days <= today`

When marked complete, the clock resets from **today** (not the window start).
One-off tasks (`frequency_days = null`) disappear once completed within the season.

## Google Sheets Setup (One-Time)

1. Create a Google Cloud project and enable the Sheets API
2. Create a service account and download the JSON key
3. Share your Google Sheet with the service account email
4. Add the key path and sheet ID to a `.env` file in `/backend`

We'll do this together at the start of Phase 2.
